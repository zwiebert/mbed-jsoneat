1 # JsoNeat
2 
3  Some C++ classes and templates to read JSON into objects in a flexible and robust way.
4 
5  The actual parsing work is done by [JSMN](https://github.com/zserge/jsmn).
6 
7  The class is a template which allows json to be of type `char *`, `const char *` or others (?).
8  There is a class to create the parser on stack.
9 
10 #### How to add to a project
11 
12 <details> <summary>Using in ESP-IDF Project</summary>
13 
14  * The component(s) are located in subdirectory `./components`
15  * The required external components are located in subdirectory `./external`
16  * I place this repository into project subdirectory `./external`. Then I use file(GLOB ..)
17  to add all components directories to `EXTRA_COMPONENTS_DIR` like so:
18  * `file(GLOB extra_dirs external/*/components external/*/external/*/components)`
19  * `list(APPEND EXTRA_COMPONENTS_DIR ${extra_dirs})`
20  * `REQUIRE ... jsmn ...`
21 
22 </details>
23 
24 <details> <summary>Using in CMake Project</summary>
25 
26  * source-CMakeLists.txt: `target_link_libraries(main PUBLIC ... jsmn ...)`
27  * project-CMakeLists.txt: `add_subdirectory(external/jsmnit/components/jsmn)`

29 </details>

38 #### Usage
39  * `#include` [jsoneat/jsoneat.hh](components/jsmn/include/jsoneat/jsoneat.hh) to use wrapper classes.
40  * `#include jsmn/jsmn.h` to use underlying Jsmn submodule directly at some places.
